On Complexity

Tools that do not degrade
with scale.

Computer programmers as we know them today have existed for less than a century. They are not credentialed or licensed. They do not sit an exam.

And yet the things entrusted to computer programs are not somehow less important than the things entrusted to professionals who do. A Boeing 747 does not stay aloft because it is required by law to do so. It stays aloft because it is governed by computer programs that make it able to do so. How this is possible is worth understanding.

Over a remarkably short period, programmers have developed tools, practices, and protocols for building large-scale systems that interoperate with one another. iPhones are not less useful when three billion people have one. The tools do not degrade with scale.

Laws, by contrast, become harder to use when the number of people who must apply them increases. It is not obvious that this limitation reflects some fundamental constraint. It may simply reflect a gap in tooling.

Print to Word. Print to PDF. Generate a redline. That is not three things. It is one thing, three times.

Prices are projections. Redlines are projections. We add bases.

Architecture
01

XPC Services

Document conversion and redline generation run in dedicated XPC services — separate processes with independent memory spaces, managed by launchd, created and destroyed on demand. Each service does one thing. The app coordinates.

Multi-Process
02

Parallel Redlining

The redline engine implements a word-level Myers diff. Multiple redlines execute in parallel — each with its own NSXPCConnection, its own handler, its own coordinated file access via NSFileCoordinator.

Structured Concurrency
03

Zero-Copy IPC

No document content crosses the XPC boundary. The app passes file paths; the services read, process, and write directly to the filesystem. The main process never holds conversion artifacts in memory.

XPC Services
NSXPCConnection, NSXPCListener. Each service runs in its own process with its own memory space. Created by launchd, destroyed on invalidation.
File Coordination
NSFileCoordinator with the batch prepare API. Coordinated reads and writes across processes without deadlock.
Multi-Format Export
Pandoc runs over XPC. Markdown in, HTML and PDF and DOCX out. One source, every format.
Structured Concurrency
Swift task groups orchestrate parallel XPC connections. Each redline job gets its own connection, its own handler, its own lifecycle.
SwiftData
Structured filing data persisted with @Model. Template fields resolve at render time against live model state.
TextKit 2
NSTextView backed by NSTextStorage. Every keystroke produces a timestamped EditTransaction, encrypted with AES-GCM on save.

Privacy by Architecture

The architecture makes the wrong thing hard to do.

Attorney-client privilege is not a feature. It is a constitutional obligation. Every tool a lawyer uses must be evaluated against it. XPC services run in separate processes with separate memory. No document content crosses the process boundary — only file paths. Edit transactions are encrypted with AES-GCM before they touch disk. There is no server. There is no API call. There is nothing to intercept, subpoena, or breach — because the data never leaves the machine.
