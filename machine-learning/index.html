The Dataset

Why regulatory filings are
a frontier ML problem.

Regulatory data is interesting not because it is law, but because of what it demands from a learning system. The SEC's EDGAR database contains millions of filings spanning decades. The Federal Register adds hundreds of pages daily. Every document is necessarily public. And none of it is clean.

The internal structure is expressed through inconsistent HTML, CSS classes that vary by filing agent, nested tables used for layout rather than data, and inline XBRL tags embedded at arbitrary depths. The structure is real — headings, sections, definitions, cross-references — but it is never declared in a schema. It must be recovered from surface signals that shift across filers, across years, across administrations.

This makes the corpus a genuinely hard problem for machine learning. The data is massive, free, and semi-structured in the specific sense that matters: internal organization exists but must be inferred through statistical regularity rather than parsed from a specification. A model trained on this material must generalize across wildly inconsistent markup to recover the stable semantic structure underneath.

Redliner's ML pipeline begins with Apple's NaturalLanguage framework. Every piece of text passes through NLTokenizer for word-level segmentation, then AtlasMLManager for cleaning and classification, then NLGazetteer for dictionary-based entity lookup — all on-device, all within the application process, all growing with your usage.

Mathematical Foundations

The Mathematics

Why this domain rewards
mathematical depth.

Structure
Topology
Document structure forms directed acyclic graphs: sections contain subsections, definitions reference defined terms, cross-references create edges between nodes. Recovering this graph from flat HTML is a topological inference problem — the model must learn which containment and adjacency relationships are structural versus accidental.
Variation
Manifolds
Filing templates vary smoothly across law firms and filing agents. A 10-K from Wachtell looks different from one prepared by Donnelley, but both inhabit a low-dimensional manifold of "valid 10-K structures." The model must learn this manifold — the space of plausible document shapes — to distinguish meaningful variation from noise in formatting.
Clustering
Spectral Methods
Eigendecomposition of document similarity matrices reveals natural clusters: risk factors group together, compensation tables cluster separately from financial statements. Spectral methods let the system discover these groupings without supervised labels, using the eigenvectors of a Laplacian constructed from pairwise document similarity.
Composition
Generating Functions
SEC filings are compositional: a prospectus is built from a cover page, a summary, risk factors, financial statements, and legal opinions, each of which has its own internal grammar. Generating functions formalize this recursive structure, encoding how smaller templates compose into larger documents and enabling the model to reason about parts and wholes.
Training
Convergence
Training on heterogeneous data — where a risk factor paragraph and a financial table have radically different statistical properties — creates convergence challenges. The loss landscape contains saddle points and narrow valleys. Guaranteeing that gradient descent reaches a useful minimum, rather than oscillating between document types, requires careful analysis of convergence conditions.
Approximation
Chebyshev Polynomials
Field distributions across filings — word counts, section lengths, entity frequencies — are not Gaussian. Chebyshev polynomials provide efficient, near-minimax approximations of these irregular distributions, enabling compact representations of statistical profiles without assuming a parametric form. This matters when the model must quickly estimate whether a document segment is typical or anomalous.
Information
Fisher Information Matrices
Not all parameters in a model are equally constrained by the data. The Fisher information matrix measures, for each parameter, how much the observed data actually tells you about its value. In this domain, some parameters are heavily constrained (the model sees thousands of "risk factor" headings) while others are barely identified (rare filing types). Fisher information reveals which parts of the model are well-grounded and which are guessing.
Curvature
Hessians
The Hessian matrix — the matrix of second derivatives of the loss function — describes the curvature of the loss landscape. Flat directions correspond to parameters the data does not constrain; sharp directions correspond to parameters where small changes cause large errors. For semi-structured data, the Hessian tends to have a few very large eigenvalues (structural features) and many near-zero ones (formatting noise), and understanding this spectrum guides optimizer design.
Prediction
Innovation Matrices
When processing a filing sequentially — paragraph by paragraph — each new block either confirms or surprises the model's expectations. The innovation matrix captures the structure of these prediction errors. In a well-calibrated system, innovations should be white noise; systematic patterns in the innovation sequence indicate that the model is failing to capture some regularity in how document sections follow one another.
Dimensionality
Overdetermined Systems
A single SEC filing contains hundreds of observable surface features — font sizes, CSS classes, tag nesting depth, word frequencies — but the latent structure it expresses may have only a dozen dimensions (filing type, preparer, era, industry). The system is massively overdetermined: far more measurements than unknowns. This is advantageous for inference but requires techniques like pseudoinverse computation and regularization to extract the latent signal cleanly.
Temporal
Non-Stationary Time Series
Regulatory language evolves. The Sarbanes-Oxley Act changed how risk factors are written. Dodd-Frank altered disclosure requirements. Plain English initiatives reshaped sentence structure. A model trained on filings from 2005 will misclassify filings from 2024 unless it accounts for non-stationarity — the fact that the underlying data-generating process is itself changing over time, and that yesterday's patterns are not tomorrow's.
Regime
Drift Terms
Beyond gradual evolution, filing patterns exhibit systematic drift tied to regulatory administrations. An SEC chair who emphasizes disclosure produces measurably different filing behavior than one who emphasizes deregulation. These drift terms are not noise — they are deterministic shifts in the data distribution that a robust model must detect and adapt to, rather than treating them as increased variance.
Capabilities
01

Semantic Tagging

AtlasTagger scans NSTextStorage for the custom attribute key .atlasSemanticTag and extracts tagged chunks with their ranges, content, and visual colors. Tags include definition, crossReference, partyName, date, amount, condition, and obligation — the semantic building blocks of a legal document.

NLTagger + TextKit 2
02

Document Classification

AtlasMLManager records every tagging event as a TrainingEntry with fields for label, humanLabel, cleanText, and context. These entries train an NLGazetteer that classifies document segments by SEC filing type — risk factors, definitions, cross-references — without ever leaving the device.

NLGazetteer + JSONL
03

Named Entity Recognition

The Tag model classifies entities into types: partyName, date, amount, condition, obligation. Each tag carries its NSRange so the editor can highlight entities in place. The MLSidebar surfaces all recognized entities in a navigable list alongside the document.

On-Device NER

The parser does not guess at structure. It reads the HTML the SEC actually produces.

EDGAR filings, parsed into typed blocks. On your machine.

SEC Document Understanding
Structured
data from
EDGAR HTML.
SECHTMLParser
Compiles a paragraph regex at load time. Extracts attribute soup and content from every <p> tag in the filing. Classifies by CSS class: H1, H2, H3, Article, Section.
SECBlockType
Eight typed block variants: heading1, heading2, heading3, paragraph, definition, listMarker, listItem, unknown.
SECBlock Metadata
Each block carries an optional [String: String] metadata dictionary for inline XBRL tags like ix:name. Codable and Sendable for safe cross-boundary use.
SECDocument
The top-level model: a Codable, Sendable array of SECBlock values. Converts to and from JSON instantly. The entire document is a value type.
List Detection
Identifies bullet points by scanning for the HTML entity &#8226; and the literal bullet character. Cleans artifacts with regex before classification.
Tag Stripping
Internal HTML tags — <span>, <b>, <i> — are removed by regex. Non-breaking spaces normalized. Content trimmed. Only clean text reaches the model.

On-Device by Design

Models run on YOUR machine. Data stays on YOUR machine.

There is no server. There is no API key. There is no telemetry endpoint. The NaturalLanguage framework ships with macOS. The training data is written to a local JSONL file in your Documents directory. The gazetteer is built in-process from that local file. The entire ML pipeline — tokenization, classification, entity recognition, tag extraction — executes within the application sandbox. No data is sent anywhere. No data can be sent anywhere. The architecture does not contain a network path.
