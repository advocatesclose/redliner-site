The Pipeline

How Metal replaces
CPU-bound text layout.

Traditional text engines measure glyphs on the CPU, one paragraph at a time. NSLayoutManager walks the glyph stream sequentially, computing line breaks, measuring advances, positioning each character. For a ten-page brief, this is fine. For a hundred-thousand-paragraph corpus, it is a wall.

Redliner's MetalTextLayout framework eliminates the sequential bottleneck. Instead of asking the CPU to measure and position glyphs, the engine pre-rasterizes the glyph set into a GPU-resident texture atlas. Each character becomes a texture coordinate. Layout becomes arithmetic: advance widths are summed, line breaks are computed, and the entire paragraph set is measured in a single pass.

The renderer packs glyph positions into vertex buffers and submits them to the Metal render pipeline. One draw call per frame. The GPU samples each glyph from the atlas texture and composites the final image. No NSTextView. No NSTextStorage for rendering. No TextKit in the hot path.

The text buffer itself is backed by a piece table data structure. Edits are splice operations on buffer ranges -- insertions append to an add buffer, deletions split existing pieces. The piece table feeds the layout engine, which feeds vertex buffers, which feed the GPU. The entire pipeline runs without blocking the main thread.

The MetalEditorView is an NSView subclass that owns the Metal device, command queue, and render pipeline state. MetalEditorRepresentable bridges it into SwiftUI via NSViewRepresentable, with bidirectional text binding and a coordinator guard against feedback loops. The result is a GPU text editor that drops into any SwiftUI view hierarchy with a single line of code.

Every glyph is a textured quad. Layout is arithmetic. One draw call per frame.

No TextKit. No NSLayoutManager. No compromise.

MetalTextLayout
Custom Swift package providing the GPU text layout pipeline. Glyph atlas generation, vertex buffer packing, and render pipeline state -- all Metal-native.
MetalEditorView
NSView subclass that owns the MTLDevice, MTLCommandQueue, and MTLRenderPipelineState. Handles keyboard input, cursor positioning, and frame rendering.
Piece Table Buffer
Text storage backed by a piece table with BufferRange-based splice operations. Append-only add buffer. O(log n) edits. Feeds the layout engine directly.
MetalEditorRepresentable
NSViewRepresentable bridge into SwiftUI. Bidirectional text binding with coordinator guard against feedback loops. One line to embed a GPU editor.
Glyph Atlas
Pre-rasterized MTLTexture containing the full glyph set. Each character is a texture coordinate lookup. No Core Text in the render loop.
Apple Silicon
Unified memory architecture means zero copies between CPU and GPU. Vertex buffers and texture atlases share the same physical memory. Built for M-series chips.
